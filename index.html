Deploy To DO

'Deploy to DO' service

The ‘Deploy to DO' button enables users to deploy apps to DigitalOcean without leaving the web browser, and with little or no configuration. The button is ideal for customers, open-source project maintainers or add-on providers who wish to provide their customers with a quick and easy way to deploy and configure any personal or any open-source application on the github.

Installation

Step One

Visit our website on http://67.205.151.140/. Insert the github URL of the "app.json" in repository wherein you want to install the Deploy to DO button.

Step Two

Click on 'Create my DO button'. This will generate a unique script for your Deploy to DO button.

Step Three

Copy the generated script and paste it in your README file on your repository or your official website. The 'Deploy to DO' button will appear in its place.

Useful Tip:

The button is well-suited for use in README files, and is intended to serve as a replacement to a list of manual steps typically required to configure an app.

How it Works?

The 'Deploy to DO' service is a handy and easy solution to configure and deploy an application on the DigitalOcean platform. Here's how the 'Deploy to DO' button will work.

Step One

Click on 'Deploy to DO' button. This will take you to the create droplet page.

Step Two

Configure your droplet according to your needs. Create new droplet once satisfied. This step will cost you credits on your digitalOcean account!

Step Three

Following the configuration, a new droplet will be created and the project will be deployed to the created droplet. You will then be redirected to your list of droplets. The user may now view the application deployed by clicking on the newly created droplet's address.

Requirements

The basic requirements for creating a button are that your app has a valid app.json file in its root directory, and that the app source code is hosted in a GitHub repository.

Understading The app.json


app.json app.json is a manifest file which provides information to deploy your app to Digital Ocean .This file describes about the image ,application and the post deploy commands of your application
Here’s a sample app.json file:


name


Contains your application name to be deployed on DigitalOcean platform. It takes lowercase alphabets (a-z), digits (0-9) ; any special characters or space are not accepted.
For Example: 


                                        {
                                            "name": "lamp Sample"
                                        }

image


It can contain any of the supported images for the droplet being created. Get list of support images here 


                                        {
                                            "image": "lamp"
                                        }

app


Copy the generated script and paste it in your README file on your repository. The 'Deploy to DO' button will appear in its place. 


                                        {
                                            "app": "lavarage",
                                        }

description


Contains the description of your application. 


                                        {
                                            "description": "A barebones Node.js app using Express 4",
                                        }

repository


It contains the link to your github repository. You must copy and paste it from the github repository that you want to create the deploy button for. Do not put any trailing slashes. 


                                        {
                                            "repository": "https://github.com/heroku/node-js-sample",
                                        }

Final app.json


Finally your app.json should look like 


                                    {
                                    "name": "Node.js Sample",
                                      "image": "",
                                    "app": "https://node-js-sample.herokuapp.com/node.png",
                                    "description": "A barebones Node.js app using Express 4",
                                  "repository": "https://github.com/heroku/node-js-sample",
                                  "post_deploy": ["node", "express", "static"]
                                    }
